# Gmail API Audit P0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve the three P0 findings of the 2026-05-16 Gmail API audit — H-1 (scope reduction), H-3 (HMAC anti-spoof of system-notification detection with 30-day legacy grace), M-1 (PSR-6 access_token cache) — delivered as three sequential commits on `main`.

**Architecture:**
- M-1 wraps the `Google\Client` with a filesystem-backed PSR-6 cache so the access_token persists across `GmailService` instances and consecutive requests (~1h TTL).
- H-3 introduces a new `App\Service\Util\NotificationStamp` class. Outbound notifications get a `[#<ticket>·s=<8-hex>]` suffix derived from `Security.salt`. Inbound check accepts that stamp as canonical, falls back to the legacy `X-Mesa-Ayuda-Notification` header **only** when `Authentication-Results` reports `dkim=pass header.d=<own domain>`, plus the existing `From == system_email` heuristic. Also removes the dead-code subject-prefix check (`Re: [Ticket #` — never matched).
- H-1 prunes `gmail.readonly` + `gmail.send` from the OAuth scope list, leaving only `gmail.modify` (which subsumes the other two for every API call this codebase makes).

**Tech Stack:** CakePHP 5.x, PHP 8.5+, `google/apiclient ^2.19.3`, new dependency `cache/filesystem-adapter:^1.2` (transitively pulls `league/flysystem`), PHPUnit suite under `tests/TestCase/`, ruleset `phpcs.xml`, static analysis via `vendor/bin/phpstan`.

**Spec:** `docs/superpowers/specs/2026-05-16-gmail-audit-p0-design.md`

### Task 7: Implement `NotificationStamp`

**Files:**

- [ ] **Step 1: Create the file**

```php
<?php
declare(strict_types=1);

namespace App\Service\Util;

use Cake\Core\Configure;

/**
 * Anti-spoof stamp embedded in outgoing notification subjects.
 *
 * Format appended at the END of the subject:
 *   " [#<ticketNumber>·s=<8-hex-HMAC>]"
 *
 * The 8 hex chars = 32 bits of HMAC-SHA256(input='ticket:<N>', key=Security.salt)
 * truncated. This is anti-spoof at the cost of one preimage chance in ~4e9 — not
 * a cryptographic signature. The salt is private; an external sender cannot mint
 * a valid stamp for an arbitrary ticket number.
 *
 * Receiver flow (GmailService::isSystemNotification):
 *   $ticket = NotificationStamp::verifiedTicketNumber($subject);
 *   if ($ticket !== null) { // treat as system reply, skip ingest }
 */
final class NotificationStamp
{
    private const STAMP_RE = '/\[#(\d+)·s=([0-9a-f]{8})\]/u';
    private const STAMP_LENGTH = 8;

    public static function append(string $subject, string $ticketNumber): string
    {
        return rtrim($subject) . ' [#' . $ticketNumber . '·s=' . self::compute($ticketNumber) . ']';
    }

    /**
     * Returns the ticket number iff the subject carries a stamp whose HMAC
     * matches the one we would compute for that same ticket number under
     * the current Security.salt. Returns null otherwise.
     */
    public static function verifiedTicketNumber(string $subject): ?string
    {
        if (!preg_match(self::STAMP_RE, $subject, $m)) {
            return null;
        }
        return hash_equals(self::compute($m[1]), $m[2]) ? $m[1] : null;
    }

    private static function compute(string $ticketNumber): string
    {
        $salt = (string)Configure::read('Security.salt');
        return substr(hash_hmac('sha256', 'ticket:' . $ticketNumber, $salt), 0, self::STAMP_LENGTH);
    }
}
```

        $systemEmail = $this->getSystemEmail();
        if ($systemEmail !== '' && strtolower($fromEmail) === strtolower($systemEmail)) {
            return true;
        }

        // Removed (dead code): subject-prefix "Re: [Ticket #" check. Templates
        // produce "Tu ticket #N fue creado", never "[Ticket #N]", so this branch
        // never matched in production.
        return false;
    }

- [ ] **Step 3: Add the private `dkimPassesForOwnDomain` helper**

Immediately after `isSystemNotification`, add:

```php
    /**
     * True iff the Authentication-Results header (added by Gmail itself on
     * inbound mail) reports dkim=pass for our own domain. Used by the H-3
     * grace-window branch to authenticate the legacy X-Mesa-Ayuda-Notification
     * header.
     *
     * @param array $headers Array of header objects from Gmail API
     */
    private function dkimPassesForOwnDomain(array $headers): bool
    {
        $authResults = $this->getHeader($headers, 'Authentication-Results');
        $systemEmail = $this->getSystemEmail();
        $atPos = strrpos($systemEmail, '@');
        if ($authResults === '' || $atPos === false) {
            return false;
        }
        $ownDomain = substr($systemEmail, $atPos + 1);

        return (bool)preg_match(
            '/dkim=pass[^;]*?header\.d=' . preg_quote($ownDomain, '/') . '\b/i',
            $authResults,
        );
    }
```
